# Мониторинг обновлений ЕМИСС

Скрипт для автоматического мониторинга выбранных показателей ЕМИСС/Fedstat и отправки уведомлений в Telegram при изменении данных.

## Что делает проект

Проект регулярно проверяет набор показателей на сайте ЕМИСС:

* загружает данные показателей в формате SDMX;
* очищает динамические поля XML, чтобы не реагировать на технические изменения;
* считает стабильный hash данных;
* сравнивает текущий hash с предыдущим состоянием;
* отправляет уведомление в Telegram, если данные показателя изменились;
* отправляет технические алерты в отдельный Telegram-чат при ошибках доступа;
* отвечает на команду `/start` списком отслеживаемых показателей.

## Требования

* Python 3.10+
* Telegram-бот, созданный через `@BotFather`
* Доступ к `https://fedstat.ru`
* Установленные Python-зависимости:

  * `requests`
  * `python-dotenv`

## Установка

Клонировать репозиторий:

```bash
git clone <URL_РЕПОЗИТОРИЯ>
cd <ПАПКА_ПРОЕКТА>
```

Создать виртуальное окружение:

```bash
python -m venv .venv
```

Активировать окружение.

Для Windows PowerShell:

```powershell
.venv\Scripts\Activate.ps1
```

Для Linux/macOS:

```bash
source .venv/bin/activate
```

Установить зависимости:

```bash
pip install requests python-dotenv
```

Опционально, если используется SOCKS-прокси:

```bash
pip install "requests[socks]"
```

## Настройка переменных окружения

Создать файл `.env` в корне проекта:

```env
TELEGRAM_BOT_TOKEN=123456789:AAxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
TELEGRAM_CHAT_IDS=123456789,987654321,-1001234567890
ALERT_CHAT_ID=123456789
```

Где:

* `TELEGRAM_BOT_TOKEN` — токен Telegram-бота от `@BotFather`;
* `TELEGRAM_CHAT_IDS` — список получателей уведомлений об обновлениях данных, через запятую;
* `ALERT_CHAT_ID` — отдельный чат для технических ошибок.

ID групп и супергрупп Telegram часто начинаются с `-100`.

## Telegram-команды

### `/start`

Команда `/start` отправляет пользователю список отслеживаемых показателей со ссылками на Fedstat. Бот ответит списком показателей и их ID.

## Как работает проверка изменений

Для каждого показателя скрипт запрашивает данные по адресу:

```text
https://fedstat.ru/indicator/<ID>.do?format=sdmx
```

Затем из XML удаляются динамические поля:

* `<Prepared>`
* `<Extracted>`
* `<ID>`

После этого считается MD5-хеш очищенного XML.

Если хеш отличается от сохраненного ранее значения, скрипт считает, что данные обновились, и отправляет уведомление в Telegram.

## Файл состояния

Состояние хранится в файле:

```text
emiss_state.json
```

В нем сохраняются последние известные хеши по каждому показателю.

При первом запуске уведомления об изменениях не отправляются: скрипт только фиксирует начальное состояние.

Удаление `emiss_state.json` приведет к повторной фиксации начального состояния.

## Интервалы проверки

Основной интервал задается в коде:

```python
CHECK_INTERVAL = 600
CHECK_INTERVAL_JITTER = 120
```

Это означает:

* базовая проверка — раз в 600 секунд;
* случайное отклонение — плюс/минус 120 секунд.

Итоговый интервал между циклами проверки будет примерно от 8 до 12 минут.

Между запросами отдельных показателей также используется случайная пауза от 3 до 8 секунд.

## Технические алерты

Если показатель недоступен или запрос к Fedstat завершается ошибкой, скрипт отправляет алерт в `ALERT_CHAT_ID`.

Чтобы не спамить одинаковыми ошибками, используется cooldown:

```python
ERROR_ALERT_COOLDOWN = 3600
```

То есть по одному показателю технический алерт отправляется не чаще одного раза в час.

## Прокси для Telegram

Желательно прокинуть доступ к Telegram через зарубежный сервер, особенно если сервер, на котором запущен мониторинг, нестабильно работает с Telegram API напрямую.

Для этого можно поднять SOCKS5-прокси через SSH-туннель. В отдельном окне PowerShell выполнить:

```powershell id="yhlw2s"
ssh -N -D 1080 user@IP
```

Где:

* `user` — пользователь на зарубежном сервере;
* `IP` — IP-адрес зарубежного сервера;
* `1080` — локальный порт SOCKS5-прокси.

После запуска этой команды локально будет доступен SOCKS5-прокси:

```text id="k5j6d9"
socks5h://localhost:1080
```

В коде в блоке `Настройки` для этого предусмотрен параметр:

```python id="275kdh"
TELEGRAM_PROXIES = None

# Раскомментить, если используется прокси через зарубежный сервер
# TELEGRAM_PROXIES = {
#     "http":  "socks5h://localhost:1080",
#     "https": "socks5h://localhost:1080",
# }
```

Если SSH-прокси поднят, нужно раскомментировать словарь:

```python id="my8jfi"
TELEGRAM_PROXIES = {
    "http":  "socks5h://localhost:1080",
    "https": "socks5h://localhost:1080",
}
```

Если прокси не используется или SSH-туннель не поднят, словарь прокси должен быть закомментирован, а значение должно остаться таким:

```python id="2ntfit"
TELEGRAM_PROXIES = None
```

Важно: если оставить активным блок:

```python id="eu447r"
TELEGRAM_PROXIES = {
    "http":  "socks5h://localhost:1080",
    "https": "socks5h://localhost:1080",
}
```

то он переопределит `TELEGRAM_PROXIES = None`. В этом случае скрипт будет пытаться отправлять Telegram-запросы через `localhost:1080`. Если SSH-туннель не запущен, отправка сообщений в Telegram будет падать с ошибкой подключения.

Для работы SOCKS-прокси нужно установить поддержку SOCKS:

```bash id="84zxfo"
pip install "requests[socks]"
```

Или добавить в `requirements.txt`:

```txt id="ur4jt6"
requests[socks]
python-dotenv
```

## Запуск на сервере через systemd

Пример unit-файла:

```ini
[Unit]
Description=EMISS Telegram Monitor
After=network.target

[Service]
WorkingDirectory=/path/to/project
ExecStart=/path/to/project/.venv/bin/python /path/to/project/monitor_v3.py
Restart=always
RestartSec=10
EnvironmentFile=/path/to/project/.env

[Install]
WantedBy=multi-user.target
```

Сохранить файл, например:

```text
/etc/systemd/system/emiss-monitor.service
```

Затем выполнить:

```bash
sudo systemctl daemon-reload
sudo systemctl enable emiss-monitor
sudo systemctl start emiss-monitor
```

Проверка статуса:

```bash
sudo systemctl status emiss-monitor
```

Просмотр логов:

```bash
journalctl -u emiss-monitor -f
```

## Возможные проблемы

### `KeyError: 'TELEGRAM_BOT_TOKEN'`

Не задана переменная окружения `TELEGRAM_BOT_TOKEN`.

Проверьте наличие файла `.env` и его содержимое.

### Бот не отправляет сообщения

Проверьте:

* правильность `TELEGRAM_BOT_TOKEN`;
* правильность `TELEGRAM_CHAT_IDS`;
* написал ли пользователь боту первым;
* добавлен ли бот в группу;
* есть ли у бота права отправлять сообщения.

### Не работает SOCKS-прокси

Установите зависимости:

```bash
pip install "requests[socks]"
```

И проверьте, что локальный прокси действительно запущен.

### Слишком много алертов об ошибках

Увеличьте значение:

```python
ERROR_ALERT_COOLDOWN = 3600
```

Например, до 3 часов:

```python
ERROR_ALERT_COOLDOWN = 10800
```
